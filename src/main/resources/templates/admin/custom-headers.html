Custom Headers

Inject custom HTML or JavaScript into the <head> tag of all pages. Use this to add analytics scripts, custom stylesheets, or other head-level integrations.

Custom HTML / JavaScript
The content is injected verbatim inside the <head> tag of every page. Leave blank to disable custom headers.
When unchecked, the custom headers are only injected into public-facing pages (dashboard, resource detail, outages, announcements).
Notes
  • The content is not sanitised — only admins can configure this setting.
  • Changes take effect immediately on the next page load.
  • Typical use cases: analytics tags, custom fonts, <style> overrides.
  • For security, avoid injecting untrusted third-party scripts.
