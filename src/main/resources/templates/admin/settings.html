General Settings

Public Access Settings
When disabled, unauthenticated users are redirected to the login page and all pages require a signed-in user.
When enabled, unauthenticated users can add resources via the API.
When enabled, unauthenticated users can manually trigger a check on any resource detail page.
When disabled, users must be authenticated to execute instant checks.
When enabled, full resource URLs are shown. When disabled, only the resource name is displayed.
Instant Check
Shows an inline check form in the dashboard header without storing a resource.
Matches configured auth credentials using the same wildcard behavior as resource checks.
Use * to allow all targets (default). Wildcards are supported, for example example.com/*.
Check History Retention
Deletes old check records in the background without blocking regular checks.
Outage Retention
Deletes closed outage records in a dedicated scheduled run using outage end date as the retention reference.
When enabled, all outage records for a resource are deleted together with it. When disabled, outage history is kept but no longer linked to any resource.
API CORS Allowed Origins
